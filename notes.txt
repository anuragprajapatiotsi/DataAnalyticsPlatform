📁 app/ (The Heart of the App)
This folder contains all your Pages and Routes. Next.js uses "File-based Routing," meaning the folder name becomes the URL.

(dashboard)/
: The parentheses mean this is a Route Group. It doesn't appear in the URL, but it allows all pages inside (Home, Explore, Settings) to share the same 
layout.tsx
 (Sidebar and Topbar).
observability/: A nested route. Folders inside like data-quality/ create URLs like /observability/data-quality.
layout.tsx
: Defines the "frame" of your site (like the Header/Sidebar) that stays the same while you navigate.
page.tsx
: The actual content of the page you see at a specific URL.
📁 components/ (The Building Blocks)
Instead of writing 1,000 lines of code in one file, we break the UI into small, reusable pieces here.

ui/: "Dumb" components like Button, Input, or 
Tabs
. They don't know about your data; they just look good and handle clicks.
catalog/ & observability/: "Smart" components specific to those features, like the 
AssetSidebar
 or 
QualityMetrics
.
sidebar.tsx
 & topbar.tsx: The main navigation elements used in the layout.
📁 utils/ (The Toolbox)
Contains helper functions that aren't UI-related but are used everywhere.

cn.ts: A very common utility in modern React used to merge CSS classes (Tailwind) intelligently.
📁 public/ (Static Assets)
Anything the browser needs to download directly without processing.

otsi-logo.png: Images, icons, and fonts go here. You can reference them in code starting with /.
📁 styles/ or Root CSS Files
globals.css: This is where Tailwind CSS is initialized and where you define your "Custom Scrollbars" or global background gradients.
📄 Key Configuration Files
package.json: The "Manifest." It lists every library (like lucide-react for icons) your project depends on.
tailwind.config.ts: The "Style Guide." Defines your colors, spacing, and animations.
next.config.js: Settings for how Next.js should build and run your app.
